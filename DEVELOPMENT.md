吞噬模拟器 - 可爱治愈版
===============================

项目结构说明
-----------

/devourer-simulator/
├── index.html          - 游戏主HTML文件，包含UI布局和样式
├── game.js            - 游戏核心逻辑（400+行代码）
└── README.md          - 详细游戏说明和操作指南

功能实现清单
-----------

✅ 角色系统
  - 6个初始可玩角色（小猫、小兔、小狐狸、黑熊、老虎、苍鹰）
  - 2个隐藏角色（金钱豹、火凤凰）需要通过击杀敌人解锁
  - 每个角色有独特的属性（攻击、防御、速度、HP）

✅ 战斗系统
  - 回合制自动战斗（靠近敌人自动触发）
  - 基于速度的先手权计算
  - 基于防御的伤害减免
  - 随机伤害浮动（增加策略性）

✅ 升级系统
  - 击杀敌人获得经验
  - 经验值呈指数增长（每级需求 = 50 × 1.15^(level-1)）
  - 升级时生命值恢复
  - 自动检测升级并显示技能选择

✅ 技能系统
  - 10种不同的技能可供选择
  - 常见技能（攻击+5、防御+4等）
  - 稀有技能（强化版本，效果更好）
  - 传奇技能（高收益高风险的混合技能）
  - 每次升级从3个随机技能中选择

✅ 2D地图系统
  - 1000x700像素开放地图
  - 网格背景（治愈系美术）
  - 随机敌人生成
  - AI敌人自主移动

✅ AI系统
  - 敌人随机漫步
  - 定期改变方向
  - 与玩家无AI寻路（使用随机移动）

✅ 数据持久化
  - LocalStorage保存最高分
  - 隐藏角色解锁进度记录
  - 游戏关闭后数据保留

✅ UI系统
  - 角色选择界面
  - 升级技能选择界面
  - 游戏结束界面
  - 实时玩家属性显示
  - 实时游戏统计显示

核心代码架构
-----------

Character类：
  - 管理所有角色属性（攻击、防御、速度、HP、等级等）
  - 支持升级、学习技能、承受伤害
  - 绘制角色（使用emoji）

Enemy类（继承Character）：
  - AI控制的敌人
  - 实现随机移动逻辑

battle()函数：
  - 实现回合制战斗逻辑
  - 计算伤害和先手权
  - 返回战斗结果

gameLoop()函数：
  - 主游戏循环（使用requestAnimationFrame）
  - 处理输入、更新游戏状态、渲染画面

游戏难度平衡
-----------

难度递进：
  - Level 1-5：新手阶段，敌人弱
  - Level 6-10：进阶阶段，需要选择合理技能
  - Level 11+：挑战阶段，敌人很强

经验平衡：
  - 早期升级快，容易获得技能
  - 后期升级慢，需要克服更多敌人
  - 敌人经验与等级成正比

隐藏角色机制：
  - 金钱豹：击杀10个敌人解锁（中等难度）
  - 火凤凰：击杀25个敌人解锁（困难，鼓励重复游玩）

如何运行
-------

方法1：直接打开HTML
  1. 用浏览器打开 index.html
  2. 点击选择角色
  3. 开始游戏

方法2：使用本地服务器
  1. cd 到游戏目录
  2. python -m http.server 8000
  3. 访问 http://localhost:8000

支持的浏览器：
  - Chrome 60+
  - Firefox 55+
  - Safari 11+
  - Edge 79+

技术栈
-----

前端：
  - HTML5 Canvas（图形渲染）
  - Vanilla JavaScript（游戏逻辑）
  - LocalStorage（数据持久化）
  - CSS3（样式和动画）

设计模式：
  - 类继承（Character → Enemy）
  - 状态机（gameState.screen）
  - 组件模式（Character类管理单个角色）

开发者笔记
---------

  - JavaScript：~550行
